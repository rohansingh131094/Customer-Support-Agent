What this is

A fully working customer support agent for Bookly, an online bookstore — handling order status, returns, and policy questions with real-time streaming and identity verification.

The Thesis

Customers expect instant, accurate, empathetic support — every time. Human teams struggle to deliver that consistently at scale. The question isn't whether to automate. It's how.

Great CX is a
reasoning
problem —
not a rules

An agent that understands what a customer needs and figures out how to help — even when the situation is messy — is a fundamentally different thing than a bot that matches inputs to outputs.

How this build reflects it
Goals over scripts
SOPs define what to achieve and what constraints to respect — not the steps to execute. Claude figures out the how.
Minimum context
Each intent gets only the system prompt and tools it needs. Sonnet never sees tools that don't apply — reducing noise and hallucination risk.
Hard constraints only
Auth before orders. Confirmation before action. These are the lines that can't be crossed — everything else, Claude owns.

Key Engineering Decisions

Every choice made deliberately — and why.

🔩
Raw SDK — No Framework

Direct Anthropic SDK calls with no abstraction layer. Full control over the tool loop, streaming, and error handling. No framework lock-in.

⚡
Two-Model Architecture

Haiku classifies intent — fast and cheap. Sonnet executes — powerful and accurate. Right model for the right job keeps cost and latency low.

🎯
Goal-Based SOPs

Prompts define what to achieve and what constraints to respect — not how to execute step by step. Claude reasons toward the goal, handling edge cases scripts can't anticipate.

🚿
Filler Text Suppression

Text before a tool call is buffered and discarded. Only the final, informed response streams to the customer — no noise, no filler.

Scaling to Production

Six decisions that separate a demo from a system that ships.

Cost & Latency
Prompt Caching
System prompt served from cache at 90% token discount — applies to every turn, every user.
Async I/O
Non-blocking request handling — thousands of concurrent sessions on minimal threads.
Scale & Reliability
Persistent Sessions
Replace in-memory state with Redis — survives restarts, enables horizontal scaling.
RAG Knowledge Base
Vector search over policy docs — grows without code changes, handles semantic queries.
Quality & Trust
Structured Observability
Trace every intent decision, tool latency, and token spend — debug failures before customers notice.
LLM-as-a-Judge Evals
Automated eval pipeline scores responses on compliance, accuracy, and tone after every prompt change.
